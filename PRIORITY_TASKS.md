## 🚀 立即要做的 3 个关键功能

### 1️⃣ 仓位激活 API（高优先级）

**目的**：将 pending（挂单）转为 active（激活订单）

**工作内容**：
```javascript
// 新增 API 端点
PUT /api/v1/positions/:id/activate

// 请求体
{
  "entry": 45000,      // 可选：更新入场价
  "size": 0.5          // 可选：更新仓位大小
}

// 响应
{
  "id": "pos_001",
  "order_type": "active",  // 从 pending 变为 active
  "status": "open",
  "entry": 45000,
  "size": 0.5
}
```

**测试方案**：
1. 创建一个 pending 仓位
2. 调用激活 API
3. 验证 order_type 已改为 active
4. 验证 Telegram 发送激活通知

---

### 2️⃣ 仓位管理 API（高优先级）

**目的**：支持修改 TP/SL，添加备注，等

**新增端点**：
```javascript
// 更新 TP/SL
PUT /api/v1/positions/:id/update
{
  "tp": [46000, 47000],    // 分档获利
  "sl": 44500,             // 止损
  "notes": "关键支撑在44500"
}

// 获取单个仓位详情
GET /api/v1/positions/:id

// 获取仓位历史（TP/SL 变化记录）
GET /api/v1/positions/:id/history
```

**测试方案**：
1. 创建仓位
2. 更新 TP/SL
3. 验证数据库已更新
4. 验证 Telegram 通知已发送
5. 查询历史记录

---

### 3️⃣ 仓位平仓 API（高优先级）

**目的**：关闭持仓，计算 PnL

**新增端点**：
```javascript
// 平仓
POST /api/v1/positions/:id/close
{
  "exit": 46000,        // 平仓价格
  "notes": "获利了结"
}

// 响应
{
  "id": "pos_001",
  "status": "closed",
  "entry": 45000,
  "exit": 46000,
  "size": 0.5,
  "pnl": 500,          // 盈利
  "pnl_percent": 2.22  // 利润率
}
```

**测试方案**：
1. 创建并激活仓位
2. 平仓
3. 验证 status=closed
4. 验证 PnL 计算正确
5. Telegram 通知包含 PnL
