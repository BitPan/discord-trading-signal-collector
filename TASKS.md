# Discord Trading Signal Collector - 开发任务拆分

## 📌 项目概览

**项目名**：discord-trading-signal-collector  
**目标**：7×24 稳定运行的 Discord 交易信号收集和解析系统  
**预计周期**：8-12 周（取决于并行度）

---

## 🏗️ 第一阶段：项目基础 (Week 1-2)

## 🔍 第三阶段：信号解析与识别 (Week 5-6)

### Task 3.1: 直接指令识别 (Direct Command Parser)
**优先级**：P0  
**时间估算**：3-4 天  
**依赖**：Task 1.1  
**内容**：
- [ ] 创建 DirectCommandParser 类
- [ ] 支持的指令格式：
  ```
  "OPEN BTCUSD 45000 0.5 BUY"
  "CLOSE 46000"
  "TP: 46000, 47000"
  "SL: 44500"
  "size: 0.5, entry: 45000"
  ```
- [ ] 正则表达式提取
  - symbol, action, entry, size, tp, sl
- [ ] 价格和数量验证
- [ ] 置信度评分

**测试**：
- [ ] 单元测试：20+ 个不同格式的指令
- [ ] 测试用例覆盖：
  - 基础格式
  - 拼写错误容错
  - 极端数值处理

**验收条件**：
- 能识别 95% 以上的标准指令
- 价格和数量解析准确
- 非指令消息返回 null

---

### Task 3.2: 多消息聚合分析 (Multi-Message Aggregator)
**优先级**：P1  
**时间估算**：4-5 天  
**依赖**：Task 3.1  
**内容**：
- [ ] 创建 MultiMessageAnalyzer 类
- [ ] 检测消息关联性
  - 同一用户，时间间隔 < 10 分钟
  - 共同的关键词（symbol, 交易方向）
- [ ] 消息聚合策略
  ```
  Msg 1: "看 BTC，强劲支撑在 44500"
  Msg 2: "Entry at 45000, size 0.5"
  Msg 3: "TP: 46000, SL: 44500"
  
  → Aggregate into single signal
  ```
- [ ] 信息优先级排序
- [ ] 数据一致性检查（例如 entry 和 size 不能冲突）

**测试**：
- [ ] 单元测试：10+ 个多消息场景
- [ ] 测试用例覆盖：
  - 3 消息、5 消息聚合
  - 消息顺序打乱
  - 包含噪声消息

**验收条件**：
- 能正确聚合 90% 的相关消息组
- 保证聚合的准确性
- 没有误识别

---

### Task 3.3: 图片 OCR 识别 (Image OCR Module)

- [ ] 数据提取和验证

**测试**：
- [ ] 单元测试：5+ 张测试订单截图
- [ ] 测试准确率和速度

**验收条件**：
- 能正确识别 85% 以上的订单截图
- 处理速度 < 3 秒/张
- 包括错误处理和降级
